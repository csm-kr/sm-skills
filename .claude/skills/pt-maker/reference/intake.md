# 인테이크 — Grill Me (Phase 1)

슬라이드를 만들기 전에 **자료와 의도를 쥐어짜는** 단계. 바로 개요를 만들지 말고, 질문을 **한 번에 하나씩** 던진다. 답이 비면 다음 주제로 넘어가지 말고 후속 질문으로 좁힌다.

## 0. 시작 게이트 — 참고자료 또는 인터뷰
첫 질문은 항상 참고자료 여부다. 여기서 **참고자료/reference**는 발표에 실제로 참고할 PPT, PDF, 문서, 스크린샷, URL, 데이터, 손글씨, 스케치 같은 원자료를 뜻한다.

```text
참고할 PPT/PDF/문서/스크린샷/URL/데이터가 있나요?
있으면 input/에 넣거나 URL/파일명을 알려주세요. 없으면 바로 인터뷰로 들어가겠습니다.
```

받은 참고자료는 목적별로 분류해 인테이크 노트에 남긴다.

- `content` — 슬라이드 내용으로 재구성할 자료
- `evidence` — 주장 근거, 수치, 사례, 스크린샷
- `style` — 기존 덱의 톤, 밀도, 레이아웃 취향
- `visual-asset` — 원본 그대로 넣을 이미지, 스케치, 로고, 캡처
- `constraint` — 반드시 지킬 조건, 금지사항, 브랜드/법무 제약

Color Hunt 같은 팔레트 사이트는 참고자료가 아니다. 색상 변경이나 새 무드를 원할 때 `palette source`로 따로 기록하고, SKILL.md의 Color Hunt 규칙에 따라 팔레트를 찾는다.

## 1. 질문 순서
아래 순서로 묻되, 매번 하나의 질문만 한다. 사용자가 이미 답한 항목은 확인만 하고 넘어간다.

1. **목적** — 이 발표가 끝났을 때 무엇이 달라져야 하는가?
2. **타겟 독자/청중** — 누가 듣고, 무엇을 이미 알고, 무엇을 결정하는가?
3. **한 줄 메시지** — 청중이 기억해야 할 주장은 무엇인가? 주제가 아니라 주장으로 쓴다.
4. **발표 상황** — 발표 시간, 장소/채널, 발표자 역할, 평가 기준, 형식(IR/제안/보고/강의/세일즈 등).
5. **근거/데이터** — 주장을 믿게 할 숫자, 사례, 스크린샷, 고객/제품 근거가 있는가?
6. **CTA** — 발표 후 청중이 무엇을 하길 원하는가?
7. **기획 방향** — 설득형, 설명형, 보고형, 데모형, 스토리형 중 무엇이 맞는가? Claude Code가 답변을 바탕으로 추천한다.
8. **분위기/톤** — 진지함, 세련됨, 친근함, 에너지, 프리미엄, 실험적 등 어떤 감정이어야 하는가?
9. **사진/이미지 레벨** — 실사 사진, 제품 스크린샷, 아이콘/SVG, AI 일러스트, 텍스트 중심 중 어느 정도까지 넣을 것인가?
10. **제약과 금지** — 반드시 피할 표현, 색, 스타일, 주장, 민감 정보가 있는가?

## 2. 필수 추출 체크리스트
아래 4개는 빠짐없이 확보해야 다음 단계로 갈 수 있다.

- [ ] **한 줄 메시지** — 발표 전체를 관통하는 한 문장
- [ ] **타겟 청중** — 사전지식, 기대, 의사결정 권한까지 구체화
- [ ] **근거/데이터** — 확보 여부와 리서치 필요 여부 구분
- [ ] **CTA** — 발표 후 원하는 행동

## 3. 후속 질문 패턴
- 목적이 흐리면: "그 발표가 성공했다는 걸 어떻게 알 수 있나요?"
- 청중이 모호하면: "그 자리에서 결정권 가진 사람은 누구인가요?"
- 메시지가 라벨이면: "그래서 청중이 기억할 한 문장은 뭔가요?"
- 발표 형식이 애매하면: "보고인가요, 설득인가요, 투자/구매 결정을 끌어내는 자리인가요?"
- 근거가 약하면: "그걸 믿게 할 숫자나 스크린샷 하나만 꼽으면 뭔가요?"
- 비주얼 요구가 모호하면: "사진을 적극적으로 쓰는 덱인가요, 다이어그램 중심인가요, 텍스트 중심인가요?"
- 범위가 넓으면: "이번 발표에서 안 다룰 건 뭐죠?"
- 데이터가 주장과 안 맞으면: 모순을 그대로 되묻는다.

## 4. 인테이크 노트
이 단계 산출물은 다음 단계로 넘길 수 있는 짧은 브리프다.

```text
목적:
기획 방향:
발표 상황:
한줄메시지:
청중:
근거/데이터: [확보 / 리서치 필요]
CTA:
분위기/톤:
사진/이미지 레벨:
참고자료 맵:
  - content:
  - evidence:
  - style:
  - visual-asset:
  - constraint:
palette source: [없음 / Color Hunt / 사용자 지정]
미해결 질문:
```

## 규칙
- 질문은 한 번에 하나만 한다.
- 답이 비면 다음 질문으로 넘어가지 말고 좁혀 묻는다.
- 사용자가 "그냥 진행"이라 해도 필수 4항목은 최소한 가정으로라도 명문화하고 확인받는다.
- 의도/청중/메시지가 정리되기 전에는 분위기나 색상부터 확정하지 않는다.
- 인테이크 전에 `../taste-profile.md`를 읽어 톤·구조 기본값을 이미 적용한 상태로 질문한다.
