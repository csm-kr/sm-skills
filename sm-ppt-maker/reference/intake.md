# 인테이크 — grill me (Phase 1)

슬라이드를 만들기 전에 **의도와 데이터를 쥐어짜는** 단계. 적응형 인터뷰지만,
아래 **필수 4항목**을 빠짐없이 확보할 때까지 끝내지 않는다. office-hours 톤으로 압박한다.

## 필수 추출 체크리스트 (4개 다 채울 때까지 진행 불가)
- [ ] **한 줄 메시지** — 발표 전체를 관통하는 한 문장. ("주제"가 아니라 "주장")
- [ ] **타겟 청중** — 구체적으로 누구인가. 사전지식·기대·의사결정 권한.
- [ ] **근거/데이터** — 주장마다 증거(실제 스샷·수치·사례). 없으면 Phase 2 리서치로.
- [ ] **원하는 행동(CTA)** — 발표 후 청중이 무엇을 하길 바라는가.

## 적응형 후속 질문 — 답변의 빈 곳을 파고든다
- 주장에 근거가 없을 때: "그걸 믿을 근거는? 숫자/사례 하나만."
- 청중이 모호할 때("개발자들"): "그 자리에서 결정권 가진 사람은 누구? 그들이 이미 아는 건?"
- 메시지가 라벨일 때("우리 제품 소개"): "그래서 청중이 기억할 한 문장은?"
- 범위가 넓을 때: "이번 발표에서 안 다룰 건 뭐죠?"
- 데이터가 주장과 안 맞을 때: 모순을 그대로 되묻는다.

## 인테이크 노트 (이 단계 산출물 — 다음 단계로 넘김)
```text
한줄메시지: ...
청중: ...
근거/데이터: [확보 / 리서치 필요]
CTA: ...
미해결 질문: ...
```

## 규칙
- 한 번에 하나~둘씩 묻는다. 답이 비면 다음으로 넘어가지 말고 후속 질문.
- 사용자가 "그냥 진행"이라 해도 필수 4항목은 최소한 가정으로라도 명문화하고 확인받는다.
- 인테이크 전에 `../taste-profile.md`를 읽어 톤·구조 기본값을 이미 적용한 상태로 질문한다.
