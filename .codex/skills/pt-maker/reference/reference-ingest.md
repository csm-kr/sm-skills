# 레퍼런스 덱 흡수 (Phase 4 보조)

사용자가 기존 PPT/PDF/덱을 주면 **콘텐츠**와 **스타일**을 분리해 활용한다.

## 입력 형태별 읽는 법
- **PDF**: `Read`의 `pages`로 직접 읽는다(슬라이드별 텍스트·레이아웃 파악).
- **이미지/스샷 덱**: `Read`로 이미지를 보고 레이아웃·톤을 파악.
- **웹 덱(URL)**: `browse`로 열어 슬라이드별 스샷.
- **PPTX 바이너리**: 직접 파싱하지 않는다 — 사용자에게 PDF로 내보내 달라고 요청(스펙 §7).

## 콘텐츠 추출 (자료로 활용)
- 사실·수치·메시지를 인테이크 노트에 합친다(출처 = "사용자 레퍼런스").
- 그대로 베끼지 않고 이번 청중·한줄메시지에 맞게 재구성.

## 스타일 추출 → 취향 갱신 제안
- 레이아웃 패턴(표지 형태, 챕터 구분, 텍스트 밀도), 색·폰트 인상, 어투를 관찰.
- **바로 적용하지 말고** taste-profile 갱신 제안으로 연결:
  > "이 레퍼런스에서 본 패턴: ① 섹션마다 큰 숫자 표지 ② 한 줄 메시지 + 여백 많음.
  >  taste-profile §1/§2에 추가할까요? [y / 수정 / skip]"
- 확인된 스타일만 `../taste-profile.md`에 반영(이름 붙은 프리셋 또는 §1 항목). 브랜드 토큰 자유 변경 금지.

## 주의
- 사용자 원본 이미지는 생성/변형 금지, 원본 그대로 임베드(기존 규칙).
- 레퍼런스의 저작 요소(로고 등)를 무단 복제하지 않는다.
