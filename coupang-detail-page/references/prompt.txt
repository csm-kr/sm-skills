# 쿠팡 상세페이지 기획 기준

이 문서는 사용자가 제공한 `prompt.txt`의 핵심 원칙을 스킬 배포용으로 정리한 기준 문서다.
중복 문장을 줄였으며 기본 출력 규격과 생성 방식은 현재 스킬 계약을 따른다.

## 1. 목표

- 제품 정보에 근거한 쿠팡 모바일 상세페이지 이미지 10장을 만든다.
- 각 이미지는 기본 `800x2400` 세로형 PNG다.
- 10장은 독립적으로 읽히면서 이어 보면 자연스러운 구매 설득 흐름을 이룬다.
- 제품, 후킹 카피, 설명, 카드, 디테일 캡션, 비교, 사용 장면을 장의 역할에 맞게 조합한다.
- 한 장을 단순 포스터 한 장으로 끝내지 않고 3~6개 세로 정보 구역으로 설계하되, 구역마다 한 메시지만 전달한다.
- 10장 전체를 하나의 브랜드 작업처럼 보이게 한다.

## 2. 사실 입력

기획 전에 다음 정보를 확인한다.

```text
상품명 / 카테고리 / 브랜드 / 모델명·SKU / 라벨 표기
핵심 장점 1·2·3
타깃 고객 / 대표 불편 / 사용 상황 / 사용 방법
구성품 / 소재 / 색상 / 크기 / 중량 / 제조국
관리 방법 / 주의사항
검증된 차별점 / 강조할 분위기
객관적 수치·인증·시험 정보 또는 없음
금지 표현
```

입력하지 않은 사실은 문구나 시각적 암시로 만들지 않는다.

## 3. 조사 기준

- 동일 제품과 유사 제품을 각각 웹 검색한다.
- 접근 가능한 실제 상세페이지를 최소 3개 비교해 첫 훅, 설명 문단, 카드·칩, 디테일 캡션, 비교·추천·CTA, 타이포 계층과 혼잡 요소를 기록한다.
- 동일 제품은 모델·라벨·패키지·외형을 확인하고 출처와 일치 근거를 기록한다.
- 유사 제품은 고객 불편, 정보 순서, 촬영 장면, 카피 각도와 카테고리 표현을 연구하는 데만 사용한다.
- 유사 제품의 기능·수치·리뷰·인증을 현재 제품 사실로 옮기지 않는다.
- 제3자의 제품, 카피, 로고, 고유 그래픽과 상세페이지를 복제하지 않는다.

## 4. 카피 매니페스트 규칙

- 모든 문구는 자연스러운 한글이다.
- 한 이미지의 모든 문자열을 `EYEBROW`, `H1`, `BODY`, `CARD·CHIP`, `CAPTION`, `CTA` 필드로 먼저 고정한다.
- 장마다 큰 `H1`은 하나지만, 설명·장점 카드·디테일 캡션·추천 라벨을 목적에 맞게 추가한다. 전체 문구를 헤드라인과 보조 한 줄로 제한하지 않는다.
- 한 장에 3~6개 정보 구역을 두고, 각 구역에는 한 가지 주장과 한 가지 시각 근거만 둔다.
- 헤드라인은 짧고 구체적이며 1~2줄 안에 들어간다.
- 고객 불편 또는 검증된 제품 차이에서 눈에 띄는 후킹을 만든다.
- 설명은 1~3개의 짧은 문장으로 쓰고, 카드·캡션은 훑어볼 수 있게 서로 다른 정보를 담는다.
- 이미지 안에서 읽기 어려운 장문, 의미 없는 영문, 근거 없는 숫자와 작은 각주를 금지한다.
- 승인 문구를 생성 과정에서 축약·의역·추가하지 않는다.

## 5. 타이포그래피와 가시성

- 10장 모두 같은 인상의 현대적인 한국어 산세리프 한 계열을 사용한다.
- 헤드라인을 이미지에서 가장 큰 텍스트로 둔다.
- 헤드라인, 섹션 제목, 본문, 카드·캡션 순서가 즉시 보이게 한다.
- 800px 원본 기준 `H1 72~104px`, `H2 44~64px`, `BODY 32~40px`, `라벨·캡션 최소 32px`를 기본 토큰으로 사용한다.
- 800px 원본을 400 CSS px로 축소해도 모든 정보 문구가 16px 이상이 되게 한다.
- 굵기는 `H1 800~900`, `H2·카드 700~800`, `BODY 500~600`, `라벨·캡션 600 이상`으로 유지한다.
- 밝은 배경에는 짙은 글자, 사진 위에는 충분한 단색 여백 또는 고대비 오버레이를 둔다.
- 글자를 제품·카드·캔버스 경계와 겹치거나 자르지 않는다.
- 장마다 서체 인상, 굵기, 정렬과 크기 규칙을 임의로 바꾸지 않는다.

## 6. 디자인 방향

- 제품과 메시지가 먼저 보이는 깔끔하고 신뢰감 있는 광고형 레이아웃을 만든다.
- 과도한 장식보다 충분한 여백과 단순한 카드 구조를 쓴다.
- 배경, 팔레트, 조명, 그림자, 카드 모양, 아이콘과 CTA 규칙을 먼저 고정한다.
- 실제 레퍼런스는 여백, 색상 관계, 정보 위계와 촬영 문법만 참고한다.
- 상품 원본의 형태, 색상, 구성품, 라벨과 부품을 레퍼런스 스타일에 맞춰 바꾸지 않는다.

## 7. 광고 제한

- 입력하지 않은 가격, 할인율, 리뷰, 별점, 판매량, 랭킹을 만들지 않는다.
- 입력하지 않은 인증, 시험 결과, 수상, 보증과 환불 문구를 만들지 않는다.
- 치료, 완치, 회복, 개선 보장과 의학적 효능을 표현하지 않는다.
- 근거 없는 `100%`, `완벽`, `무조건`, `최고`, `1위`를 쓰지 않는다.
- 신체 변화나 효과 보장형 Before/After를 만들지 않는다.
- 쿠팡 로고, 앱 화면, 검색·장바구니·하트 아이콘, 스마트폰 목업을 넣지 않는다.
- 이미지 픽셀 안에 Page, 장 번호, 페이지 번호를 넣지 않는다.

## 8. 10장 기본 흐름

| 장 | 역할 | 권장 내용 |
|---|---|---|
| 01 | 문제 후킹 | 대표 불편, 큰 헤드라인, 공감 설명, 핵심 칩 2개 |
| 02 | 필요한 이유 | 기존 무드의 아쉬움 또는 선택 기준 카드 3개 |
| 03 | 상품 소개 | 제품 히어로, 상품명, 한 줄 약속, 확인된 장점 3개 |
| 04 | 상황·스타일 비교 | 서로 다른 착장·상황 2개와 각 장면 캡션 |
| 05 | 장점 1·2 | 실제 디테일 2~3컷, 디테일 제목과 설명 |
| 06 | 장점 3·디자인 무드 | 컬러·질감·실루엣을 보여주는 에디토리얼 설명 |
| 07 | 차별점 | 확인 가능한 시각 차이를 3~5개 항목으로 비교 |
| 08 | 추천 대상 | 서로 다른 대상·옷차림·선호 4~6개 |
| 09 | 활용 장면 | 실제 사용·스타일링 장면 4~6개와 짧은 캡션 |
| 10 | CTA | 제품 히어로, 확인된 장점 3개 요약, 중립 CTA |

상품 특성상 특정 장이 맞지 않으면 같은 설득 기능을 하는 구성으로 바꾸되 사실 범위는 유지한다.

## 9. 생성 순서

1. 입력 이미지와 상품 정보를 확인한다.
2. 동일·유사 제품을 조사하고 근거 범위를 나눈다.
3. 사실 원장과 금지 원장을 만든다.
4. 10장 카피·장면·읽는 순서와 공통 타이포그래피를 표로 기획한다.
5. 사용자의 번호 승인을 받는다.
6. 장별 `image_gen` 호출에서 제품·디자인·승인 한국어를 한 번에 완성한다.
7. 각 결과의 텍스트, 제품, 레이아웃, 광고 주장을 직접 검사한다.
8. 이상한 장만 실패 원인에 맞춰 재생성한다.
9. 10장 흐름과 `800x2400` 파일 규격을 검증한다.
10. GIF·영상이 이해를 높이는 장면을 골라 모션 아이디어와 선택적 ComfyUI 핸드오프를 만든다.

## 10. 장별 image_gen 프롬프트 골격

```text
사용 사례: ads-marketing
자산: 쿠팡 모바일 상세페이지 10장 중 N장, 페이지 번호는 렌더링하지 않음
목적과 후킹: <설득 역할과 첫 시선 메시지>
입력 역할: ORIGINAL_PRODUCT / WEB_CONFIRMED_PRODUCT / STYLE_REFERENCE
제품 불변 요소: <원본에서 확인한 형태·색상·부품·라벨·구성품>
공통 디자인: <팔레트·조명·카드·아이콘·여백·CTA>
공통 타이포: 같은 한국어 산세리프 계열, H1 72~104px, H2 44~64px, BODY 32~40px, 라벨·캡션 최소 32px, 높은 대비
장면과 구도: 800x2400, 1:3 세로 캔버스, 3~6개 세로 정보 구역과 명확한 읽는 순서
카피 직접 조판: 승인 문구를 완성 디자인 안에 정확히 렌더링하고 빈 자리표시자를 남기지 않음
정확한 카피 매니페스트: EYEBROW "..." / H1 "..." / BODY "..." / CARD·CHIP·CAPTION ["...", "..."] / CTA "..."
금지: 매니페스트 밖의 추가 카피, 깨진 한글, 32px 미만 정보 문구, 더미 영문, 허위 주장, 워터마크, 쿠팡 UI, 레퍼런스 복제
```

## 11. QA 기준

- 승인 카피와 글자 단위로 일치한다.
- 자모 깨짐, 오탈자, 중복 글자, 의미 없는 영문과 추가 문구가 없다.
- 헤드라인은 가장 먼저 보이고, 모든 본문·카드·캡션은 800px 원본에서 최소 32px로 모바일에서도 읽힌다.
- 장의 목적에 맞는 3~6개 정보 구역이 있으며, 같은 내용만 반복하거나 한쪽에 과밀하지 않다.
- 10장의 서체 인상, 굵기, 크기 관계, 팔레트와 무드가 일관된다.
- 제품 실루엣, 비율, 색상, 부품·라벨·구성품 수가 원본과 맞는다.
- 글자, 제품, 카드와 캔버스 경계가 충돌하거나 잘리지 않는다.
- 허위 사실, 금지 UI, 로고, 페이지 번호와 근거 없는 효과 암시가 없다.

실패한 장만 다시 생성한다. 텍스트 실패는 먼저 `image_gen` 직접 조판을 재시도하며, 후편집은 사용자 선택을 받은 예외 경로다.
