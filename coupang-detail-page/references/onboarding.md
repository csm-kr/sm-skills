# 첫 실행 온보딩

## 목적

처음 사용하는 사람이 폴더 구조를 모르더라도 원본 상품과 스타일 레퍼런스를 혼동하지 않고, 검증된 정보만으로 상세페이지 10장을 만들도록 대화를 안내한다.

## 절대 대화 규칙

1. 한 번의 응답에는 질문을 정확히 하나만 넣는다.
2. 선택 가능한 질문은 `1.`, `2.`, `3.` 형식으로 제시한다. 사용자가 번호 하나만 답해도 진행할 수 있어야 한다.
3. 자유 입력이 필요한 질문은 현재 질문 번호를 붙인 답변 예시 하나만 준다. 예: `5. 흰색 팔토시`.
4. 아직 묻지 않은 다음 질문을 같은 응답에 미리 붙이지 않는다.
5. 답을 받으면 저장한 값 하나를 확인하고 다음 질문 하나만 한다.
6. 파일과 이전 답변에서 이미 확인한 사실은 다시 묻지 않는다.
7. 사진으로 보이는 속성도 확정 사실로 저장하기 전에 사용자의 번호 선택으로 확인받는다.
8. 선택지에 없는 답이 오면 자유 입력으로 해석하고 필요한 경우 그 내용만 확인한다.
9. 생성 준비가 끝나기 전에는 `image_gen`을 호출하지 않는다.

## 시작 설명

첫 질문 앞에서 다음 내용을 짧게 한 번 설명한다.

- 원본 상품 사진은 `<skill-root>/inputs/<project-no>/original-images/`에 저장한다. 최소 1장, 정면·측면·디테일·구성품·착용/사용 모습을 포함한 3~8장을 권장한다.
- 참고할 실제 상세페이지 이미지는 `<skill-root>/inputs/<project-no>/real-references/`에 저장한다. 1~3장을 권장하지만 선택 사항이다. 레이아웃·색감·여백만 참고하며 그 상품, 로고, 문구와 수치는 복제하지 않는다.
- 상품 정보는 `<skill-root>/inputs/<project-no>/product-info.md`에 답변마다 저장한다.
- 결과는 장별로 10회 생성하며 후보는 `outputs/<project-no>/raw/`, 최종 10장은 `outputs/<project-no>/final/`에 `page-01.png`부터 `page-10.png`로 저장한다.
- 최종본은 모두 `780x3000`이어야 한다. 입력에 없는 효능, 가격, 리뷰, 인증과 수치는 만들지 않으며 준비 검사를 통과해야 생성한다.

설명 뒤에는 아래 순서의 현재 질문 하나만 보인다.

## 질문 순서

프로젝트 번호를 확정한 즉시 `python3 scripts/init_project.py --prepare <project-no>`를 실행한다. 이미 작성된 `product-info.md`는 보존되며, 새 설치에서는 Git에서 제외되는 런타임 양식이 생성된다.

### 1. 프로젝트 선택

기존 빈 시작 프로젝트가 있으면 다음처럼 묻는다.

```text
1번 질문. 어느 프로젝트로 시작할까요?
1. 현재 001 계속 사용
2. 새 번호 프로젝트 생성
3. 기존 다른 번호 입력
번호로 답해주세요.
```

사용자가 이미 프로젝트 번호를 지정했다면 이 질문은 건너뛴다.

### 2. 원본 상품 사진

먼저 폴더와 대화 첨부를 검사한다. 이미지가 있으면 개수와 관찰 가능한 시점만 말하고 다음 중 하나를 묻는다.

```text
2번 질문. 현재 원본 사진으로 진행할까요?
1. 그대로 진행
2. 사진을 더 추가
3. 저장 위치를 다시 안내
번호로 답해주세요.
```

이미지가 없으면 `1. 지금 첨부 / 2. 폴더 위치 다시 보기 / 3. 중단`을 제시한다. 원본이 없는 상태에서 진행 선택지는 주지 않는다. 대화 첨부를 로컬 파일로 복사할 수 없으면 대화에서만 사용한다고 정확히 알리고, 실제 첨부 개수를 생성 전 검사에 `--conversation-originals <count>`로 전달한다. 대화 전용 이미지는 최대 5장만 허용한다. 6장 이상이면 핵심 원본을 최대 5장으로 선별할지, 나머지를 로컬 폴더에 저장할지 한 질문으로 확인한다.

### 3. 실제 상세페이지 레퍼런스

레퍼런스 개수를 확인한 뒤 다음처럼 묻는다.

```text
3번 질문. 참고 상세페이지를 어떻게 할까요?
1. 넣어둔 레퍼런스로 진행
2. 1~3장 추가
3. 레퍼런스 없이 기본 디자인으로 진행
번호로 답해주세요.
```

레퍼런스는 선택 사항이며 제3자 자료를 임의로 검색하거나 다운로드하지 않는다.

### 4. 상품 정보 입력 방식

```text
4번 질문. 상품 정보를 어떻게 입력할까요?
1. 한 항목씩 질문받기
2. 작성한 product-info.md 사용
3. 정리된 상품 설명을 한 번에 붙여넣기
번호로 답해주세요.
```

첫 실행 기본값은 `1`이다. 사용자가 한 항목씩을 선택하면 아래 항목을 한 턴에 하나씩 묻는다. 각 질문에는 연속 번호를 붙이고 사용자가 `<번호>. <내용>`으로 답할 수 있는 예시를 하나 보여준다.

1. 상품명
2. 카테고리
3. 핵심 장점 1
4. 핵심 장점 2
5. 핵심 장점 3
6. 타깃 고객
7. 대표 불편
8. 사용 상황
9. 사용 방법
10. 구성품
11. 소재/재질
12. 색상
13. 사이즈, 중량, 제조국 중 제공 가능한 정보
14. 관리/세탁/보관 방법
15. 주의사항
16. 경쟁 제품 대비 검증된 차별점 또는 `없음`
17. 강조할 분위기
18. 객관적인 수치/인증/시험 정보 또는 `없음`
19. 사용하면 안 되는 표현 또는 `없음`

사진에서 후보를 제안할 수 있는 항목은 `1. 제안값 승인 / 2. 직접 수정 / 3. 정보 없음`으로 먼저 확인한다. 사진에서 확인할 수 없는 기능, 소재, 크기, 효능과 사용법은 제안하지 않는다.

## 답변 저장

- 각 답을 받은 직후 해당 줄 하나를 `product-info.md`에 저장한다.
- `없음`, `미제공`도 의도적인 답으로 저장할 수 있지만 상품명, 카테고리, 장점 3개, 타깃, 불편, 사용 상황·방법, 구성품, 소재, 색상과 주의사항은 임의로 `없음` 처리하지 않는다.
- 사용자가 이전 답을 바꾸면 기존 줄만 수정하고 나머지를 보존한다.
- 이미지 파일을 옮길 때 원본은 보존하고 같은 이름을 덮어쓰지 않는다.

## 생성 전 마지막 확인

모든 답변 뒤 `python3 scripts/check_project.py <project-no>`를 실행한다. 대화에만 존재하는 원본이 있으면 `--conversation-originals <count>`를 함께 사용한다. 실패하면 누락 항목 하나만 다음 질문으로 묻는다. 성공하면 로컬 사진 수와 대화 전용 사진 수, 레퍼런스 수, 상품명, 사용할 사실과 금지 표현, 결과 위치를 짧게 요약하고 다음 하나만 묻는다.

```text
마지막 질문. 상세페이지 10장 생성을 시작할까요?
1. 시작
2. 입력 다시 검토
3. 중단
번호로 답해주세요.
```

`1`을 받은 경우에만 장별 `image_gen` 10회 생성으로 넘어간다.
