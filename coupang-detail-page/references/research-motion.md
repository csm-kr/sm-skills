## 필수 리서치 원칙

모든 프로젝트에서 사용자 제공 레퍼런스 유무와 관계없이 웹 리서치를 실행한다. 목적은 동일 제품을 찾아 사실 확인을 보조하고, 유사 제품에서 카테고리의 고객 질문과 시각 문법을 파악하는 것이다.

- 동일 제품 검색과 유사 제품 검색을 별도로 수행한다.
- 사용자 원본은 제품 외형의 최우선 기준이다. 웹 결과가 원본을 덮어쓰게 하지 않는다.
- 동일 제품 여부와 출처 신뢰도를 서로 다른 축으로 평가한다. 신뢰도 높은 사이트의 유사 제품도 현재 상품의 사실 근거가 아니다.
- 유사 제품은 문제 상황, 콘텐츠 순서, 사용 장면과 정보 위계의 기획에만 사용한다. 외형, 사양, 수치, 효능, 인증, 리뷰와 카피를 전이하지 않는다.
- 직접 URL, 페이지 제목, 접근일, 검색어와 관찰 근거를 남긴다. 검색 결과 페이지나 스니펫만을 최종 근거로 인용하지 않는다.
- 특정 상세페이지의 문장, 레이아웃, 인물, 소품, 제품 사진, 로고와 고유 그래픽을 복제하지 않는다. 여러 자료에서 반복되는 추상 원리를 새 디자인 시스템으로 합성한다.
- 인터넷 도구를 사용할 수 없거나 결과가 없으면 시도한 검색어와 실패 상태를 기록한다. 검색을 생략하거나 성공했다고 가장하지 않는다.

## 동일성 등급 M0~M3

| 등급 | 판정 | 사용 범위 |
|---|---|---|
| `M0` | 현재 상품과 불일치하거나 동일성을 판단할 정보가 없음 | 사용하지 않거나 결과 없음으로 기록 |
| `M1` | 모델·SKU 또는 고유 라벨·패키지·외형이 일치해 동일 제품임을 확인함 | 출처 등급이 허용하는 범위에서 보조 근거로 사용 |
| `M2` | 고유 라벨과 여러 불변 요소가 일치하지만 모델·옵션 중 일부가 미확인임 | 동일 제품 유력 후보; 보이는 외형만 보조하고 사실 확정 전 사용자 원본과 재대조 |
| `M3` | 같은 카테고리·용도의 유사 제품이지만 다른 제품임 | 기획 아이디어만 허용 |

색상, 길이, 구성 수량과 패키지가 다른 변형은 같은 상품군이어도 현재 옵션 기준으로 `M1`이 아니다. 판단 근거를 짧게 문장으로 남기고 “닮아 보임”만으로 등급을 올리지 않는다.

## 출처 등급 E1~E4

| 등급 | 출처 유형 | 사용 범위 |
|---|---|---|
| `E1` | 브랜드·제조사·공공기관·인증기관의 공식 페이지, 설명서, 카탈로그 또는 원문 기록 | 문서가 정확한 옵션에 직접 명시한 사실만 보조 가능 |
| `E2` | 신뢰할 수 있는 공식 유통사·판매처의 동일 모델 페이지 | 식별 정보와 명시 사양을 교차 확인 가능 |
| `E3` | 마켓플레이스·개인 판매의 직접 상품 페이지와 실제 사진 | 보이는 외형·라벨·착용 구조만 보조; 성능·인증 근거 금지 |
| `E4` | 검색 스니펫, 집계·소셜 자료 또는 유사 카테고리 페이지 | 후보 발견과 카테고리 기획에만 사용 |

판매량, 리뷰, 가격과 할인은 현재 상세페이지의 광고 사실로 자동 승격하지 않는다. 시점과 판매처가 달라질 수 있는 정보는 사용자가 명시적으로 제공하고 사용할 범위를 승인했을 때만 쓴다.

## 근거 사용 매트릭스

- `M0`: 불일치 자료는 폐기하고 결과 없음은 검색 기록으로만 남긴다.
- `M1 + E1/E2`: 정확한 옵션이 일치할 때 명시된 제품 사실을 카피 후보로 제안할 수 있다.
- `M1/M2 + E3`: 웹 사진에 직접 보이는 라벨·색상·구성·외형·착용 구조만 교차 확인한다.
- `M2 + E1/E2`: 유력 후보로 기록하고 사용자 원본과 충돌이 없는 범위만 보조한다. 중요한 사실은 사용자 확인 전 확정하지 않는다.
- `M3 + 모든 E`: 카테고리 문제 각도, 장면, 정보 구조와 시각 문법 기획에만 사용한다.
- `모든 M + E4`: 상품 사실 근거로 쓰지 않는다.

웹 근거가 사용자 입력 또는 원본과 충돌하면 자동 병합하지 않는다. 옵션 차이인지 오기인지 조사한 뒤에도 충돌이 남으면 한 질문으로 기준을 확인한다.

## 검색과 기록 절차

1. 원본에서 상품명, 브랜드·모델, 라벨 문구, 패키지 문구, 색상, 독특한 구조를 추출한다.
2. 정확한 상품명, 따옴표로 묶은 라벨 문구, 모델명과 핵심 외형 조합으로 동일 제품을 검색한다.
3. 이미지와 페이지를 직접 열어 옵션·구성·라벨·디테일을 대조한다.
4. 같은 카테고리, 타깃과 사용 상황 조합으로 유사 제품과 실제 상세페이지를 검색한다.
5. 최소한 동일 제품 검색 결과와 유사 제품 검색 결과를 각각 기록한다. 결과가 없으면 `M0`와 검색어를 남긴다.
6. 사실 확인용 메모와 기획 전용 메모를 분리해 `outputs/<project-no>/web-research.md`에 저장한다.
7. 허용 사실을 `fact-ledger.md`에 옮길 때 직접 URL과 `M/E` 등급을 함께 연결한다. 유사 제품 내용은 사실 원장에 옮기지 않는다.

기록 양식:

```text
검색어: <query>
직접 URL: <direct URL>
페이지 제목: <title>
접근일: <YYYY-MM-DD>
동일성: M0 | M1 | M2 | M3
출처: E1 | E2 | E3 | E4
관찰 근거: <일치·불일치한 식별 요소>
사용 범위: 사실 확인용 | 기획 전용 | 사용 안 함
채택 내용: <짧은 메모>
```

## GIF와 영상 아이디어

정적 10장 기획을 마친 뒤 각 장을 검토해 움직임이 정보를 실제로 더 잘 전달하는 후보 1~3개만 제안한다. 모션은 정적 10장 납품을 대체하거나 장 수에 포함하지 않는다.

좋은 후보:

- 제품의 착용·설치·접기·열기처럼 순서가 중요한 사용법
- 소재의 살랑임, 탄성 또는 표면 변화처럼 원본과 검증 사실로 확인된 물성
- 여러 사용 상황 사이의 짧은 전환
- 제품의 360도 회전 또는 디테일 줌처럼 구조 이해에 도움이 되는 움직임
- 불편한 상황과 편리한 상황을 과장 없이 전환하는 장면

피할 후보:

- 정지 이미지에 단순 확대·축소만 반복해 정보가 늘지 않는 모션
- 확인되지 않은 냉감, 방수, 내구성, 치료·개선 효과를 암시하는 입자·온도·신체 변화
- 제품 형태, 구성 수량, 라벨이나 착용 방향이 프레임마다 변하는 장면
- 텍스트가 변형되거나 읽기 전에 사라지는 장면
- 유사 제품 영상의 움직임·배경·카피를 그대로 재현하는 장면

각 아이디어에 대상 장, 목적, 형식(`GIF` 또는 `video`), 길이, 핵심 동작, 시작·종료 프레임, 반복 방식, 필요한 원본과 금지 요소를 기록한다. GIF는 2~4초의 짧고 자연스러운 반복, 영상은 4~8초의 한 가지 핵심 동작을 기본 제안으로 사용하되 상품 특성에 따라 조정한다.

## 선택적 ComfyUI 핸드오프

사용자가 선택하고 실제 ComfyUI 연결 또는 워크플로가 제공될 때만 실행한다. 연결이 없으면 실행을 가장하지 말고 `outputs/<project-no>/comfyui-handoff.md`에 재현 가능한 핸드오프를 작성한다.

```text
MOTION_ID: <motion-01>
source still: <page or original image path>
product identity references: <paths>
format: GIF | video
canvas/aspect: <target; static page default is 800x2400>
duration: <seconds>
fps: <frames per second>
loop: none | seamless
motion prompt: <one verified action, camera locked unless needed>
negative prompt: product morphing, extra parts, color drift, label drift, text distortion, unsupported effects
first frame: <description>
last frame: <description>
seed/workflow/model: <provided values or unresolved>
output path: <planned path>
QA: <identity, motion continuity, copy legibility, loop seam>
```

ComfyUI용 프롬프트에서도 사용자 원본을 제품 정체성 기준으로 두고, 유사 제품 자료를 입력 정체성 이미지로 사용하지 않는다. 모델, 노드, 체크포인트와 라이선스가 확인되지 않으면 임의로 특정 항목을 확정하지 말고 `unresolved`로 남긴다.

## 모션 QA

- 첫 프레임과 마지막 프레임에서 제품 실루엣, 색상, 부품과 라벨이 원본과 같은가
- 프레임 사이에 제품이 녹거나 늘어나거나 구성품이 생기고 사라지지 않는가
- 움직임이 검증된 사용법·물성만 표현하는가
- 한글 카피가 모든 프레임에서 정확하고 충분히 오래 읽히는가
- GIF 반복 경계가 튀지 않고 영상 시작·종료가 의도한 상태인가
- 사용한 원본과 웹 자료의 출처·권리·사용 범위가 기록됐는가

실패한 모션만 다시 생성하거나 핸드오프를 수정한다. 정적 10장을 함께 변경하지 않는다.
