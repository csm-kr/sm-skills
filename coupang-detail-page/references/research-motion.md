## 필수 리서치 원칙

모든 프로젝트에서 사용자 제공 레퍼런스 유무와 관계없이 웹 리서치를 실행한다. 목적은 동일 제품을 찾아 사실 확인을 보조하고, 유사 제품에서 카테고리의 고객 질문과 시각 문법을 파악하는 것이다.

- 동일 제품 검색과 유사 제품 검색을 별도로 수행한다.
- 사용자 원본은 제품 외형의 최우선 기준이다. 웹 결과가 원본을 덮어쓰게 하지 않는다.
- 동일 제품 여부와 출처 신뢰도를 서로 다른 축으로 평가한다. 신뢰도 높은 사이트의 유사 제품도 현재 상품의 사실 근거가 아니다.
- 유사 제품은 문제 상황, 콘텐츠 순서, 사용 장면과 정보 위계의 기획에만 사용한다. 외형, 사양, 수치, 효능, 인증, 리뷰와 카피를 전이하지 않는다.
- 직접 URL, 페이지 제목, 접근일, 검색어와 관찰 근거를 남긴다. 검색 결과 페이지나 스니펫만을 최종 근거로 인용하지 않는다.
- 특정 상세페이지의 문장, 레이아웃, 인물, 소품, 제품 사진, 로고와 고유 그래픽을 복제하지 않는다. 여러 자료에서 반복되는 추상 원리를 새 디자인 시스템으로 합성한다.
- 인터넷 도구를 사용할 수 없거나 결과가 없으면 시도한 검색어와 실패 상태를 기록한다. 검색을 생략하거나 성공했다고 가장하지 않는다.

## 동일성 등급 M0~M3

| 등급 | 판정 | 사용 범위 |
|---|---|---|
| `M0` | 현재 상품과 불일치하거나 동일성을 판단할 정보가 없음 | 사용하지 않거나 결과 없음으로 기록 |
| `M1` | 모델·SKU 또는 고유 라벨·패키지·외형이 일치해 동일 제품임을 확인함 | 출처 등급이 허용하는 범위에서 보조 근거로 사용 |
| `M2` | 고유 라벨과 여러 불변 요소가 일치하지만 모델·옵션 중 일부가 미확인임 | 동일 제품 유력 후보; 보이는 외형만 보조하고 사실 확정 전 사용자 원본과 재대조 |
| `M3` | 같은 카테고리·용도의 유사 제품이지만 다른 제품임 | 기획 아이디어만 허용 |

색상, 길이, 구성 수량과 패키지가 다른 변형은 같은 상품군이어도 현재 옵션 기준으로 `M1`이 아니다. 판단 근거를 짧게 문장으로 남기고 “닮아 보임”만으로 등급을 올리지 않는다.

## 출처 등급 E1~E4

| 등급 | 출처 유형 | 사용 범위 |
|---|---|---|
| `E1` | 브랜드·제조사·공공기관·인증기관의 공식 페이지, 설명서, 카탈로그 또는 원문 기록 | 문서가 정확한 옵션에 직접 명시한 사실만 보조 가능 |
| `E2` | 신뢰할 수 있는 공식 유통사·판매처의 동일 모델 페이지 | 식별 정보와 명시 사양을 교차 확인 가능 |
| `E3` | 마켓플레이스·개인 판매의 직접 상품 페이지와 실제 사진 | 보이는 외형·라벨·착용 구조만 보조; 성능·인증 근거 금지 |
| `E4` | 검색 스니펫, 집계·소셜 자료 또는 유사 카테고리 페이지 | 후보 발견과 카테고리 기획에만 사용 |

판매량, 리뷰, 가격과 할인은 현재 상세페이지의 광고 사실로 자동 승격하지 않는다. 시점과 판매처가 달라질 수 있는 정보는 사용자가 명시적으로 제공하고 사용할 범위를 승인했을 때만 쓴다.

## 근거 사용 매트릭스

- `M0`: 불일치 자료는 폐기하고 결과 없음은 검색 기록으로만 남긴다.
- `M1 + E1/E2`: 정확한 옵션이 일치할 때 명시된 제품 사실을 카피 후보로 제안할 수 있다.
- `M1/M2 + E3`: 웹 사진에 직접 보이는 라벨·색상·구성·외형·착용 구조만 교차 확인한다.
- `M2 + E1/E2`: 유력 후보로 기록하고 사용자 원본과 충돌이 없는 범위만 보조한다. 중요한 사실은 사용자 확인 전 확정하지 않는다.
- `M3 + 모든 E`: 카테고리 문제 각도, 장면, 정보 구조와 시각 문법 기획에만 사용한다.
- `모든 M + E4`: 상품 사실 근거로 쓰지 않는다.

웹 근거가 사용자 입력 또는 원본과 충돌하면 자동 병합하지 않는다. 옵션 차이인지 오기인지 조사한 뒤에도 충돌이 남으면 한 질문으로 기준을 확인한다.

## 검색과 기록 절차

1. 원본에서 상품명, 브랜드·모델, 라벨 문구, 패키지 문구, 색상, 독특한 구조를 추출한다.
2. 정확한 상품명, 따옴표로 묶은 라벨 문구, 모델명과 핵심 외형 조합으로 동일 제품을 검색한다.
3. 이미지와 페이지를 직접 열어 옵션·구성·라벨·디테일을 대조한다.
4. 같은 카테고리, 타깃과 사용 상황 조합으로 유사 제품과 실제 상세페이지를 검색한다.
5. 최소한 동일 제품 검색 결과와 유사 제품 검색 결과를 각각 기록한다. 결과가 없으면 `M0`와 검색어를 남긴다.
6. 사실 확인용 메모와 기획 전용 메모를 분리해 `outputs/<project-no>/web-research.md`에 저장한다. 내려받은 웹 이미지는 검색 기록의 `ASSET_ID` 열에 Asset Registry ID를 연결한다.
7. 허용 사실을 `fact-ledger.md`에 옮길 때 직접 URL과 `M/E` 등급을 함께 연결한다. 유사 제품 내용은 사실 원장에 옮기지 않는다.

기록 양식:

```text
검색어: <query>
직접 URL: <direct URL>
페이지 제목: <title>
접근일: <YYYY-MM-DD>
동일성: M0 | M1 | M2 | M3
출처: E1 | E2 | E3 | E4
관찰 근거: <일치·불일치한 식별 요소>
사용 범위: 사실 확인용 | 기획 전용 | 사용 안 함
채택 내용: <짧은 메모>
ASSET_ID: <저장한 WEB_MATCH/REF의 Axx; 없으면 `-`>
```

## 증명형 영상 소구점

예쁜 회전·확대보다 고객의 구매 반론을 실제 화면으로 줄이는 영상을 우선한다. 후보마다 다음 항목을 먼저 잠근다.

```text
구매 반론: <고객이 믿기 어려워하거나 확인하려는 한 가지>
한정 주장: <allowed facts와 실제 촬영 조건 안에서만 말할 문장>
시험 또는 동작: <한 가지 조건·부하·행동>
보이는 판정: <시청자가 화면에서 성공·실패를 직접 가를 기준>
CAPTURE_MODE: REAL_TEST | REAL_DEMO | AI_ILLUSTRATION | NO_PROOF_HOOK
```

기본 타임 구조는 `문제·반론 1초 → 실제 제품의 한 시험·동작 2~4초 → 확인 가능한 결과 1~2초`다. 한 영상에서 여러 기능을 나열하지 않는다.

- `REAL_TEST`: 실제 동일 제품과 기록된 조건·부하를 촬영한다. 물성·성능·내구·고정력·효능처럼 시간축의 물리적 결과를 주장하려면 이 모드가 필수다.
- `REAL_DEMO`: 실제 동일 제품으로 구조·착용·조작을 보여준다. 화면에서 직접 확인되는 사용법과 외형만 설명하며 성능 보장으로 확대하지 않는다.
- `AI_ILLUSTRATION`: 생성형 영상 또는 ComfyUI로 이미 허용된 구조·사용법을 이해시키는 연출이다. 실제 시험의 증거가 아니며 실증 카피를 붙이지 않는다.
- `NO_PROOF_HOOK`: 실사 원본·조건·판정 기준이 없어 하드 게이트를 통과한 후보가 없음을 뜻한다. 장식이나 튜토리얼을 TOP1로 승격하지 않고 필요한 촬영 근거를 적는다.

- 좋은 실증: 실제 제품에 한 가지 조건을 가하고 시험 전·행동·결과가 이어져 시청자가 판정할 수 있는 장면.
- 좋은 실사 데모: 플랫 상태에서 실제 착용 완성까지, 고유 커프를 엄지에 맞추는 순서, 접힌 구조가 사용 형태로 펼쳐지는 과정.
- 조건부 실증: 동일 조건의 구조 비교. 비교 축과 차별점이 사용자 입력 또는 허용 근거에 있고 실제 비교 촬영이 있어야 한다.
- 금지 증명: 시험 근거 없는 충격·진동·낙하·파손 방지, 냉감 입자, UV 차단 광선, 방수 물방울, 신체 변화와 효과 보장.

휴대폰 거치대를 흔들리는 차량에서 보여주는 식의 극적인 데모는 실제 동일 제품, 기록된 진동·부하·설치 조건, 안전한 촬영 환경과 핵심 시험 구간의 연속 원본이 있을 때만 `REAL_TEST`로 사용한다. 촬영된 조건에서 보인 결과만 말하고 `휴대폰이 부서지지 않는다` 같은 일반적 파손 방지 보장으로 확대하지 않는다. 핵심 시험 영역이나 원인·결과를 생성형 도구로 수정한 합성물은 실사 증거가 아니다.

## GIF와 영상 아이디어

고유 정보 단위에 따라 선정한 정적 페이지 기획을 마친 뒤, 각 페이지를 검토해 움직임이 정보를 실제로 더 잘 전달하는 후보를 모은다. 모든 후보를 7요소로 점수화한 뒤 하드 게이트를 통과한 상위 3개만 스토리보드로 제안한다. 모션은 선정된 정적 페이지 납품을 대체하거나 페이지 수에 포함하지 않는다.

좋은 후보:

- 제품의 착용·설치·접기·열기처럼 순서가 중요한 사용법
- 소재의 살랑임, 탄성 또는 표면 변화처럼 원본과 검증 사실로 확인된 물성
- 여러 사용 상황 사이의 짧은 전환
- 제품의 360도 회전 또는 디테일 줌처럼 구조 이해에 도움이 되는 움직임
- 불편한 상황과 편리한 상황을 과장 없이 전환하는 장면

피할 후보:

- 정지 이미지에 단순 확대·축소만 반복해 정보가 늘지 않는 모션
- 확인되지 않은 냉감, 방수, 내구성, 치료·개선 효과를 암시하는 입자·온도·신체 변화
- 제품 형태, 구성 수량, 라벨이나 착용 방향이 프레임마다 변하는 장면
- 텍스트가 변형되거나 읽기 전에 사라지는 장면
- 유사 제품 영상의 움직임·배경·카피를 그대로 재현하는 장면

각 아이디어에 대상 장, 목적, 형식(`GIF` 또는 `video`), 길이, 핵심 동작, 시작·종료 프레임, 반복 방식, 필요한 원본과 금지 요소를 기록한다. GIF는 2~5초의 짧고 자연스러운 반복 또는 1회 재생 후 종료 프레임 유지, 영상은 4~8초의 한 가지 핵심 동작을 기본 제안으로 사용하되 상품 특성에 따라 조정한다. 착용 순서 같은 비가역 동작을 핑퐁 루프로 되감지 않는다.

## 모션 후보 7요소 점수화

각 축을 0~20점, 총 140점으로 평가해 `motion-plan.md`에 근거와 함께 기록한다.

| 평가 축 | 20점에 가까운 상태 | 낮은 점수 신호 |
|---|---|---|
| 동작 필요성 | 순서·조작·구조가 정지 사진만으로는 즉시 이해되지 않음 | 정지 확대로 같은 정보를 충분히 전달함 |
| 주장 안전성 | 원본·사용자 정보·허용 근거와 동작이 직접 연결됨 | 미확인 효능·물성·신체 변화를 암시함 |
| 전환 관련성 | 구매 전 핵심 질문을 해결하고 다음 카피·CTA로 자연스럽게 연결됨 | 예쁘지만 제품 이해·구매 판단에 영향이 작음 |
| 제작 실행성 | 선택한 `CAPTURE_MODE`에 맞는 실물·촬영 환경 또는 생성 입력과 안전한 제작 경로가 준비됨 | 필요한 실물·조건·원본·제작 경로가 없음 |
| 제품 안정성 | 제품 실루엣·색·부품·라벨을 마스크·포즈·딥스로 잠그기 쉬움 | 제품이 크게 휘거나 가려져 프레임 변형 위험이 큼 |
| 고유 소구 선명도 | 이 제품의 확인된 특징이 한 동작으로 구분되고 구매 이유가 즉시 보임 | 다른 제품으로 바꿔도 같은 영상이 성립하거나 장식에 그침 |
| 실증 후킹력 | 구매 반론을 반증 가능하게 시험하고 조건·행동·결과가 보여 시청자가 판정할 수 있음 | 사용법 설명·분위기 연출에 그치거나 결과를 검증할 수 없음 |

점수 구간은 `0~5 부적합`, `6~10 정지 우선`, `11~15 조건부 후보`, `16~20 강한 후보`를 기본 앵커로 삼는다. 총점은 우선순위 보조이며 안전성을 대체하지 않는다.

- `주장 안전성` 하드 게이트: 허용 사실 밖의 효능이나 신체 변화를 암시하면 총점에 관계없이 탈락시킨다.
- `제품 안정성` 하드 게이트: 핵심 부품·라벨·구성 수량을 잠글 방법이 없으면 후보를 정지 이미지로 돌린다.
- `고유 소구 선명도` 하드 게이트: 12점 미만이면 TOP1로 선정하지 않는다.
- `실증 후킹력` 하드 게이트: 12점 미만이면 TOP1로 선정하지 않는다. 이 점수는 총점이 보상할 수 없다.
- `REAL_TEST` 하드 게이트: 실제 동일 제품, 시험 조건·부하, 화면 판정 기준, 핵심 구간의 연속 원본 경로와 편집 내역이 없으면 물리적 성능 주장을 탈락시킨다.
- `REAL_DEMO` 하드 게이트: 구조·착용·조작처럼 화면에서 직접 보이는 사실만 허용한다. 물성·내구·고정력·효능 주장에는 사용할 수 없다.
- `AI_ILLUSTRATION` 하드 게이트: `실증 후킹력`은 최대 5점이며 TOP1이 될 수 없다. `테스트`, `입증`, `버틴다`, `된다` 같은 실증 표현과 결과 배지를 금지한다.
- 통과 후보가 없으면 `CAPTURE_MODE: NO_PROOF_HOOK`으로 기록하고 필요한 실사 제품·조건·샷을 적는다.
- 총점과 하드 게이트를 통과한 후보 중 구매 반론을 가장 크게 줄이는 하나를 `가장 영상이 필요한 소구점`으로 명시한다.

점수와 캡처 모드를 채운 뒤 `python3 scripts/validate_motion.py <project-no>`를 실행한다. 실패하면 TOP1을 승인하거나 증명형 영상으로 납품하지 않는다.

## TOP3 스토리보드와 삽입 순서

하드 게이트를 통과한 상위 3개를 `video-plan.md`에 `motion-01`~`motion-03`으로 기록한다. 점수가 같으면 `실증 후킹력 → 주장 안전성 → 고유 소구 선명도 → 동작 필요성 → 제품 안정성`이 높은 순으로 선정한다.

각 TOP3 스토리보드에 다음을 모두 포함한다.

1. `CAPTURE_MODE`, 구매 반론, 한정 주장과 상세페이지 삽입 위치
2. 시험 조건·부하·통제와 시청자가 확인할 보이는 판정
3. 실제 촬영 원본 경로와 핵심 시험 구간의 연속성·컷·속도 변경 내역; AI이면 `AI_ILLUSTRATION` 표시
4. GIF·MP4 형식, 캔버스, 길이, FPS, 루프·음소거 상태
5. 초 단위 장면 타임라인과 구간별 정확한 고정 카피
6. 시작·종료 프레임, 제품 불변 요소와 선택한 모드의 제작 지시
7. 산출 경로, 증거 일치·프레임 일관성·카피 가독성·루프 경계 QA

상세페이지 삽입 순서는 `문제·상황 정지 장 → 동작으로 풀어주는 모션 → 세부 근거·요약 정지 장`을 기본으로 삼는다. 영상 다음 정지 장이 같은 카피를 반복하면 영상의 문구를 줄이고 동작 이해에 집중한다.

## 고정 폰트 레이어

`font-plan.md`에 실제로 사용할 한글 서체 계열, 굵기, 크기, 자간, 행간, 안전 여백, 라이선스와 조판 환경을 잠그다.

- 원본 모션 플레이트에는 광고 카피를 생성하지 않는다.
- 승인한 한국어 카피를 실제 폰트 레이어로 한 번 조판한 뒤 전체 클립에 고정 합성한다. 프레임마다 재생성하지 않는다.
- 글자가 장면을 따라야 하면 텍스트 레이어 자체에 일관된 트래킹을 적용하되 글자 형태·자폭·줄바꿈은 변경하지 않는다.
- 제품에 원래 붙은 물리 라벨은 광고 카피가 아니다. 원본 텍스처를 제품에 트래킹하고 새 문자로 재생성하지 않는다.
- 대체 서체를 사용하면 자폭과 굵기가 달라지므로 모든 줄바꿈·안전 여백·표시 시간 QA를 다시 한다.

## 선택적 ComfyUI 실행·핸드오프

모션 TOP3는 기획 단계에서 먼저 만들되, `plan-review.html`을 사용자가 승인한 뒤 한 응답에 하나씩 정확한 `motion-NN`, 지원되는 GIF·VIDEO 범위, CAPTURE_MODE에 맞는 환경을 선택받는다. [approval-generation.md](approval-generation.md)의 질문과 상태 매핑을 따른다.

`AI_ILLUSTRATION`은 사용자가 선택하고 `CONNECTED` ComfyUI의 생성 시점 live 확인까지 성공할 때만 렌더한다. `CONNECTED`는 `python3 scripts/comfyui_receipt.py probe <project-no> --endpoint <http(s) URL>`로 실제 `/system_stats`를 호출해 만든 24시간 이내 receipt가 필요하며 생성 게이트가 같은 endpoint를 다시 probe한다. `WORKFLOW_PROVIDED`는 `python3 scripts/comfyui_receipt.py workflow <project-no> --workflow <project-local.json>`로 구조가 유효한 ComfyUI graph와 SHA256을 묶은 핸드오프 상태이지 실제 렌더 권한이 아니다. 자세한 계약은 [comfyui-receipt-schema.md](comfyui-receipt-schema.md)를 따른다. 실제 렌더를 포기하고 검증된 워크플로까지 넘기면 `STATIC_PLUS_MOTION_HANDOFF / WORKFLOW_PROVIDED`, 워크플로 없이 넘기면 `STATIC_PLUS_MOTION_HANDOFF / HANDOFF_ONLY`를 기록한다. 별도 `comfyui-handoff.md`를 작성하면 `video-plan.md`에 경로를 연결하고 상태를 중복 표시한다.

실제 동일 제품을 촬영하는 `REAL_TEST`·`REAL_DEMO`는 ComfyUI 없이 `NOT_REQUIRED`로 제작할 수 있다. 단, FFmpeg `ffprobe`로 RAW_TEST·RAW_DEMO 클립의 decode 가능 여부, dimensions, duration, frames를 모두 확인해야 하며 도구나 값이 없으면 fail-closed한다.

`HANDOFF_ONLY`는 기획·프롬프트·스토리보드가 완성되었지만 실제 클립 생성, 렌더, 프레임 QA는 실행하지 않았다는 뜻이다. 이 상태에서 산출 GIF·MP4가 있는 것처럼 보고하지 않는다.

실제 렌더 직전에는 `python3 scripts/validate_generation_gate.py <project-no> --target motion`, 인계 확정 직전에는 `--target handoff`를 실행한다. 기획 승인, ComfyUI 상태 또는 모션 승인이 비어 있으면 진행하지 않는다.

ComfyUI와 다른 생성형 영상은 항상 `CAPTURE_MODE: AI_ILLUSTRATION`이다. 제품 구조와 사용 순서를 이해시키는 보조 연출로만 쓰며 실제 시험 원본, `REAL_TEST` 또는 성능 증거로 표기하지 않는다. 실사 클립에 생성 배경을 합성하더라도 시험 대상·부하·접촉·결과 영역을 생성 수정하면 증거 자격을 잃는다.

```text
MOTION_ID: <motion-01>
CAPTURE_MODE: AI_ILLUSTRATION
source still: <page or original image path>
product identity references: <paths>
format: GIF | video
canvas/aspect: <target; static page default is 800x2400>
duration: <seconds>
fps: <frames per second>
loop: none | seamless
motion prompt: <one verified action, camera locked unless needed>
negative prompt: product morphing, extra parts, color drift, label drift, text distortion, unsupported effects
copy layer: FIXED_TEXT_LAYER; <font family/path, exact strings, positions and timing>
first frame: <description>
last frame: <description>
seed/workflow/model: <provided values or unresolved>
output path: <planned path>
QA: <identity, motion continuity, copy legibility, loop seam>
```

ComfyUI용 프롬프트에서도 사용자 원본을 제품 정체성 기준으로 두고, 유사 제품 자료를 입력 정체성 이미지로 사용하지 않는다. 모델, 노드, 체크포인트와 라이선스가 확인되지 않으면 임의로 특정 항목을 확정하지 말고 `unresolved`로 남긴다.

## 모션 QA

- `CAPTURE_MODE`가 실제 제작 방식과 일치하고 실사·AI 여부를 숨기지 않았는가
- `REAL_TEST`는 실제 동일 제품, 조건·부하·판정 기준, 원본 경로와 핵심 시험 구간의 연속성이 기록됐는가
- 실사 시험의 컷·속도 변경·합성 내역이 공개되고 결과를 바꾸는 생성 수정이 없는가
- REAL_TEST·REAL_DEMO RAW 클립의 decode·dimensions·duration·frames가 `ffprobe`로 확인됐는가
- 영상 카피가 촬영된 조건에서 직접 보인 결과만 말하고 일반적 성능 보장으로 확대하지 않는가
- `AI_ILLUSTRATION`이 실증 표현·결과 배지 없이 이해용 연출로만 표시됐는가
- 첫 프레임과 마지막 프레임에서 제품 실루엣, 색상, 부품과 라벨이 원본과 같은가
- 프레임 사이에 제품이 녹거나 늘어나거나 구성품이 생기고 사라지지 않는가
- 움직임이 검증된 사용법·물성만 표현하는가
- 한글 카피가 모든 프레임에서 정확하고 충분히 오래 읽히는가
- 고정 텍스트 레이어의 서체·자폭·줄바꿈·위치가 시작·중간·종료 프레임에서 일치하는가
- 제품의 물리 라벨이 없어지거나 새 글자로 바뀌지 않고 원본 텍스처로 추적되는가
- GIF 반복 경계가 튀지 않고 영상 시작·종료가 의도한 상태인가
- 사용한 원본과 웹 자료의 출처·권리·사용 범위가 기록됐는가

실패한 모션만 다시 촬영·생성하거나 핸드오프를 수정한다. 증거 무결성 실패를 AI 재생성으로 통과시키지 않으며 승인된 정적 페이지를 함께 변경하지 않는다.
