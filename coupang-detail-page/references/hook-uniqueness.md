# 문제 훅·정보 비중복·증명형 영상 규칙

## 목적

고정 장수를 채우기 위해 역할 이름만 바꾼 비슷한 포스터를 만들지 않는다. 하나의 독점 정보 단위는 구매 질문 하나에 답하고 페이지 하나를 소유한다. 영상은 제품 고유 특징을 한 동작으로 이해시키거나 실제 조건에서 판정하게 한다.

## 첫 페이지 문제 공감 훅

첫 페이지 `PROBLEM_HOOK`은 제품 설명보다 고객의 실제 상황을 먼저 보여준다. 상품에 맞는 훅 유형 하나만 고른다.

- 경험 질문형: `이런 경험 있으셨나요?`
- 상황 직시형: 확인된 불편을 구체적인 생활 장면으로 보여주기
- 선택 기준형: 고객이 실제로 비교할 기준을 한 문장으로 제시하기
- 장면 전환형: 불편 장면에서 제품이 필요한 이유로 짧게 연결하기

경험 질문은 고정 카피가 아니다. 검증된 불편이 있고 다음 문장이 현재 상품과 구체적으로 이어질 때만 쓴다. 공포, 질병, 신체 변화와 근거 없는 실패 상황을 만들지 않는다.

## 기능 우선 게이트

- 가장 중요한 기능적 구매 이유와 근거 Fact ID를 먼저 고정한다.
- 첫 훅은 그 기능이 답하는 불편과 연결한다. `PRODUCT_INTRO`는 그 기능과 상품 정체성을 한 번만 소개한다.
- 색·코디·감성·무드는 기능 이해와 선호 판단을 돕는 보조 소구다.
- 근거 있는 기능이 없으면 만들지 않는다. 관찰 가능한 구조, 실측, 구성품, 사용법 또는 관리 정보로 대체한다.
- 사용자 지정 훅은 그대로 쓸 수 있지만 수치·지속·보장 표현을 덧붙이지 않는다.

`plan-gate.md`에는 `핵심 기능 소구 / 기능 근거 Fact ID / 기능이 답하는 구매 불편 / 디자인 보조 소구 / 기능 없음 사유 / 기능 우선 상태`가 있어야 한다.

## 가변 장수 정보 계약

장수는 `3~10장`, 보통 `5~8장` 안에서 고르며 `정보 단위 수 = 선정 장수`여야 한다. 입력 사실이 부족하면 3~4장으로 줄인다. 고정 장수를 맞추기 위해 새 주장을 만들거나 같은 사실을 다른 말투로 반복하지 않는다.

각 후보를 아래 형식으로 작성한다.

```text
PAGE_ID: <연속 번호>
ROLE_ID: <역할>
INFO_ID: <이 페이지가 독점하는 정보 단위>
구매 질문: <고객이 이 페이지에서 답을 얻는 한 가지 질문>
독점 Fact ID: <다른 페이지가 주제로 소유하지 않는 근거>
필수 시각 증거: <카피를 화면에서 확인시키는 원본·도해·실측>
새 정보: <직전 페이지에 없던 구매 판단 정보>
연결 문장: <주장을 반복하지 않고 다음 질문으로 넘기는 문장>
```

강제 규칙:

1. 첫 페이지는 `PROBLEM_HOOK`이다.
2. `PRODUCT_INTRO`는 정확히 한 페이지만 둔다.
3. 마지막 정보 페이지에는 중립 CTA를 함께 둔다. `CTA`만을 위한 독립 페이지는 만들지 않는다.
4. 각 페이지는 고유한 `INFO_ID`, 구매 질문, 독점 Fact ID와 필수 시각 증거를 갖는다.
5. 같은 논리를 문제·비교·추천·활용·요약 형태로 바꾼 후보는 삭제하거나 한 페이지로 병합한다.
6. 비교, 추천 대상, 사용 장면은 각각 새로운 검증 정보가 있을 때만 페이지가 된다. 카피만 새로우면 페이지가 아니다.
7. 실측, 정확한 사양, 구성품, 사용법, 관리·세탁·보관과 주의사항처럼 구매 판단을 바꾸는 정보는 우선한다.
8. 실측은 현재 제품을 자로 잰 사용자 값·자 사진 또는 정확한 동일 옵션의 `M1 + E1/E2` 공식 사양만 허용한다. 원본 사진 비율, 유사 제품, 리뷰와 생성형 추정값은 금지한다.
9. 독립 페이지끼리 같은 PRIMARY_FACT를 소유하지 않는다. 인접 페이지는 공통 Fact ID를 최대 하나만 공유한다.
10. 제품명·법정 또는 필수 주의 문구 외 동일한 4어절 이상 문장을 반복하지 않는다.

허용 가능한 역할 예시는 `FIT_STRUCTURE`, `FEATURE_EVIDENCE`, `MEASURED_SIZE`, `COMPONENTS`, `OPTIONS`, `HOW_TO_USE`, `CARE_GUIDE`, `SAFETY_GUIDE`, `SITUATION_USE`, `NEUTRAL_COMPARE`, `GUIDE_CTA`, `OPTIONS_CTA`다. 이는 필수 목록이 아니다. 검증 정보가 있는 역할만 선택한다.

## 정보 단위 삭제·병합 판정

두 후보가 아래 중 하나에 해당하면 별도 페이지로 두지 않는다.

- 구매 질문의 답이 사실상 같다.
- 독점 Fact ID가 없고 다른 페이지의 요약만 한다.
- 같은 제품 컷과 같은 캡션을 교환해도 의미가 유지된다.
- 추천 대상 또는 활용 상황이 앞 페이지의 기능을 다시 말할 뿐 새 판단 기준을 주지 않는다.
- 중립 비교가 현재 상품의 검증 차별점을 추가하지 않는다.
- 마지막 CTA가 새 정보 없이 앞 상품 소개를 복제한다.

`plan-gate.md`에 `정보 단위 수 / 선정 장수 / 장수 결정 근거 / 삭제·병합 역할 / 장수 결정 상태`를 기록한다. `장수 결정 상태: 완료`가 아니면 HTML 승인이나 생성을 요청하지 않는다.

## 제품 컷과 레이아웃 비중복

- 모든 제품 컷에 `SHOT_ID`, 장면에 `SCENE_ID`, 지배 모듈에 `LAYOUT_ID`를 붙인다.
- 원본 한 장의 동일하거나 유사한 크롭을 여러 페이지의 주 시각으로 쓰지 않는다.
- 새 착용 장은 의상, 행동, 배경, 각도·크롭, 조명 중 최소 세 요소가 이전 착용 장과 달라야 한다.
- 같은 모델·의상·세트·포즈 계열을 두 페이지 이상의 주 시각으로 반복하지 않는다.
- 2×2 그리드, 세로 3단, 비교표, 주석형 실루엣, 매크로 모자이크와 단계 타임라인은 각각 최대 한 번만 쓴다.
- 인접 페이지는 같은 지배 모듈을 쓰지 않는다. 디자인 토큰은 통일하되 페이지 실루엣은 구분한다.
- 물리 라벨을 의도적으로 확대하는 페이지는 하나만 허용한다.
- 정규화를 위한 무의미한 상·하단 패딩은 완성 높이의 10%를 넘기지 않는다.

## 기획 검증

```bash
python3 scripts/validate_plan.py <project-no>
python3 scripts/validate_asset_map.py <project-no>
```

스크립트는 선정 장수 범위, 첫 훅, 단일 상품 소개, 마지막 정보+CTA, 필수 셀과 명시적 식별자 중복을 검사한다. 에이전트는 자동 검사 뒤에도 INFO_ID, Fact 집합, 카피, 제품 컷과 전체 contact sheet를 직접 대조한다.

검증을 통과하면 `build_plan_review.py`로 선정한 모든 페이지의 주제·카피·근거·식별자가 담긴 HTML을 만든다. 사용자가 HTML을 확정하기 전에는 생성 게이트를 승인하지 않는다. 장수, INFO_ID, 카피 또는 시각 근거가 바뀌면 기존 승인을 무효화한다.

## 증명형 영상 소구점

영상은 장식적 확대·회전보다 제품 고유 특징으로 구매 반론 하나를 화면에서 판정하는 장면이어야 한다.

```text
구매 반론: <고객이 확인하려는 한 가지>
한정 주장: <허용 근거와 촬영 조건 안에서만 말할 문장>
시험 또는 동작: <한 가지 조건·부하·행동>
보이는 판정: <시청자가 성공·실패를 직접 가를 기준>
CAPTURE_MODE: REAL_TEST | REAL_DEMO | AI_ILLUSTRATION | NO_PROOF_HOOK
```

권장 구조는 `문제·반론 1초 → 실제 제품의 시험·동작 2~4초 → 확인 가능한 결과 1~2초`다.

- `REAL_TEST`: 실제 동일 제품과 기록된 조건·부하를 촬영한다. 물성·성능·내구·고정력·효능 주장에 필수다.
- `REAL_DEMO`: 실제 제품의 끼우기, 펼치기, 접기와 같은 보이는 구조·사용법만 설명한다.
- `AI_ILLUSTRATION`: 허용 사실을 이해시키는 생성형 연출이다. 실제 시험 또는 성능 증거가 아니다.
- `NO_PROOF_HOOK`: 실사 근거와 판정 기준이 없어 하드 게이트를 통과한 후보가 없음을 뜻한다.

충격·진동·냉감·UV·방수·내구와 신체 변화는 실제 동일 제품, 기록된 조건과 판정 가능한 연속 원본이 없으면 영상으로도 입증하지 않는다.

## 영상 실증·고유성 게이트

`동작 필요성 / 주장 안전성 / 전환 관련성 / 제작 실행성 / 제품 안정성 / 고유 소구 선명도 / 실증 후킹력`을 각 0~20점으로 평가한다.

- `고유 소구 선명도` 또는 `실증 후킹력`이 12점 미만이면 TOP1이 될 수 없다.
- `주장 안전성` 또는 `제품 안정성`을 통과하지 못하면 총점과 관계없이 탈락한다.
- `AI_ILLUSTRATION`의 `실증 후킹력`은 최대 5점이며 `테스트·입증·버틴다·된다` 카피를 금지한다.
- 통과 후보가 없으면 `NO_PROOF_HOOK`과 필요한 실사 촬영을 기록한다.
- 통과한 모션은 해당 구매 질문의 정적 페이지 앞이나 뒤에 연결한다. 새 정보 없이 같은 카피를 반복하지 않는다.

## 최종 비중복 QA

- 전체 contact sheet에서 각 페이지 실루엣이 즉시 구분되는가?
- `PAGE_ID / ROLE_ID / INFO_ID / 구매 질문 / 독점 Fact ID / SHOT_ID / SCENE_ID / LAYOUT_ID`를 대조했는가?
- 각 페이지가 다른 페이지에 없는 구매 판단 정보를 하나씩 주는가?
- 비교·추천·활용·요약·독립 CTA가 반복을 만들지 않아 삭제 또는 병합됐는가?
- 첫 훅, 단일 상품 소개와 마지막 정보+CTA가 서로 다른 질문에 답하는가?
- 실측과 사양이 허용된 현재 상품 근거에서만 나왔는가?

하나라도 실패하면 `UNIQUENESS=RETRY`로 기록하고 해당 페이지를 다시 기획한다.
