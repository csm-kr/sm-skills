전체 흐름

각 게이트를 통과해야 다음 단계로 넘어갑니다. 기획 HTML 승인 전에는 정적 이미지도 만들지 않고, ComfyUI 상태와 모션 승인이 없으면 GIF·영상을 실행하지 않습니다.

01

한 번에 한 질문으로 입력 수집

원본, 레퍼런스, 상품 정보와 선택 사항을 번호로 확인합니다. 마지막 보기는 항상 ‘다른 답변하기’입니다.

Input gate
02

RAW·근거·레퍼런스 직접 확인

대화 첨부도 번호 프로젝트에 저장하고 SHA256을 기록한 뒤에만 증거로 씁니다. 원본은 제품 정체성, 실측·실사는 증거, 웹 확인 이미지는 보조, 레퍼런스는 추상적 스타일 기준으로 분리합니다.

Identity gate
03

동일 제품 + 여러 상세페이지 조사

일치 근거와 출처 강도를 기록하고, 최소 3개 실제 페이지의 훅·본문·카드·캡션 패턴을 비교합니다.

Evidence gate
04

기능·실측 우선, 최소 충분 장수 결정

한 장마다 독점 INFO_ID·Proof·구매 질문을 부여하고, 반복되는 비교·추천·요약은 삭제해 3~10장 중 필요한 수만 남깁니다.

Plan gate
05

plan-review.html 검토

모든 장의 주제·카피·화면·RAW·Proof·레퍼런스 원리·주장 경계와 모션 후보를 한 HTML에서 보여주고 사용자 확정을 받습니다.

Human gate
06

정확한 모션 ID·형식·환경 확인

기획 승인 뒤 한 질문씩 motion-NN, GIF·VIDEO 범위, REAL 촬영, live CONNECTED 실행 또는 WORKFLOW_PROVIDED·HANDOFF_ONLY 인계를 정확히 선택합니다.

Environment gate
07

정적·모션·최종 승인 분리

기획 소스와 HTML 파일 해시를 잠그고 정적, GIF·영상 또는 인계, 최종 실행 승인을 각각 받은 대상만 generation gate로 보냅니다.

Permission gate
08

텍스트까지 image_gen에서 직접 생성

한 장씩 독립 호출해 제품·디자인·승인 한국어를 함께 완성합니다. 정적 글자 2회 실패 뒤 사용자 승인 때만 고정 텍스트 대체를 허용합니다.

Generation
09

의미 QA + INFORMATION GAIN

TEXT·PRODUCT·LAYOUT·CLAIMS·ROLE·SOURCE_LINEAGE·UNIQUENESS와 각 장이 실제로 새 정보를 주는지 검사합니다.

Meaning gate
10

800 × 2400 파일 검증

정적은 Pillow 자동 runtime·안전한 fallback으로 연속 번호, dimensions, 손상·중복을 검사하고 정규화한 뒤 다시 시각 검수합니다.

File gate

온보딩 원칙

한 응답, 한 질문

사용자가 이미 준 답을 다시 묻지 않습니다. 매 답변 뒤 저장한 한 항목만 확인하고 다음 질문으로 이동합니다.

선택권을 닫지 않기

추천안을 표시할 수 있지만 강요하지 않습니다. 모든 선택 질문의 마지막 번호는 언제나 다른 답변하기입니다.

작업이 보이게 만드는 진행 멘트

진행 중 업데이트 없는 상태가 60초를 넘지 않게 합니다. 각 멘트에는 완료한 것, 지금 하는 것, 다음 할 것을 담습니다.

관문별 업데이트

입력 확인 → 웹 조사 → 정보 단위·장수 결정 → 생성 → QA·재생성 → 정규화·납품을 지날 때마다 상태를 알립니다.

장별 생성 현황

N/선정 장수 생성 시작을 먼저 알리고, 저장·검수 후 N/선정 장수 PASS 또는 RETRY와 다음 장을 안내합니다.

실패도 즉시 공개

깨진 한글, 잘못된 제품, 작은 글자와 과장 표현을 숨기지 않고 실패 항목과 재시도 방향을 설명합니다.

짧고 검증 가능하게

막연한 “진행 중” 대신 확인한 파일 수, 조사한 페이지 수, 생성·통과한 장 수처럼 검증 가능한 상태를 말합니다.

동일·유사 제품 리서치

동일성과 출처 신뢰도를 분리하고 여러 페이지의 구조를 비교하면, ‘같아 보이는 사진’이 검증된 기능 사실로 잘못 승격되는 것을 막으면서 실제 상세페이지의 정보 밀도는 배울 수 있습니다.

판정의미사용 범위
M1 + E1/E2모델·SKU 또는 고유 라벨과 공식/신뢰 판매처가 일치명시된 사실을 카피 후보로 제안 가능
M1/M2 + E3외형과 라벨은 일치하나 마켓·개인 판매 출처외형·착용 구조 보조 확인만
M3 + E4같은 카테고리의 유사 제품문제 각도·장면·정보 구조 아이디어만
M0불일치 또는 동일 제품 확인 못함결과 없음 자체만 기록

인기 수치 경계: 순위·최근 구매량·리뷰 수는 조사일의 사례 선정 근거입니다. 현재 상품의 판매 카피나 성능 근거로 옮기지 않습니다.

R2P: RAW에서 PAGE까지 끊기지 않는 계보

RAW → REFERENCE → REASON → PROOF → PAGE 순서로 잠급니다. 디자인 레퍼런스나 AI 생성본이 제품 사실로 승격되는 순간 검증은 실패합니다.

  • 서체 변화·작은 각주·낮은 대비·겹침 금지

  • GIF·영상 카피는 항상 고정 레이어로 후합성
  • 제품 물리 라벨은 원본 텍스처로 트래킹

기획 HTML을 먼저 확정하기

plan-review.html은 핵심 기능 다음에 장별 카피·화면·RAW·Proof·레퍼런스 원리·주장 경계와 모션 TOP3를 보여줍니다. 사용자 승인 때 HTML의 소스 해시와 승인한 HTML 파일 자체의 SHA256을 함께 기록하며, 둘 중 하나라도 현재 파일과 다르면 생성이 자동 차단됩니다.

1. 기획 검토 + 해시 2종

build_plan_review.py로 HTML을 만든 뒤 실제 응답·날짜, data-source-digest와 승인한 HTML 파일 SHA256을 기록합니다.

2. 정확한 motion-NN

검증된 후보의 정확한 ID와 CAPTURE_MODE를 번호로 선택합니다. 정적 전용만 선정 모션 ID: 해당 없음입니다.

3. GIF·VIDEO 범위

선정 ID가 지원하는 형식만 보여주고 STATIC_PLUS_GIF, STATIC_PLUS_VIDEO, STATIC_PLUS_GIF_VIDEO 또는 STATIC_PLUS_MOTION_HANDOFF를 잠급니다.

4. 환경 + helper 증빙

REAL_TEST·REAL_DEMO 실사는 NOT_REQUIRED가 가능하고, AI 실제 렌더는 receipt와 생성 시점 live 재확인을 모두 통과한 CONNECTED만 허용합니다. WORKFLOW_PROVIDED는 구조가 검증된 ComfyUI 워크플로 인계 전용이며 실제 렌더 완료를 뜻하지 않습니다. 수기·generic JSON은 무효입니다.

5. 승인 세 단계

정적 승인, GIF·영상 또는 인계 준비 승인, 최종 실행 승인을 각각 다른 질문과 기록으로 남깁니다. 최종 토큰은 SCOPE·MOTION·COMFY를 모두 잠그며 환경이 바뀌면 다시 승인받습니다. 마지막 보기는 언제나 다른 답변하기입니다.

6. 대상별 하드 게이트

--target static, motion, handoff를 분리합니다. HANDOFF_ONLY는 실제 영상 렌더를 통과할 수 없습니다.

CLAIMS

허위 기능·수치·UI·로고·페이지 번호와 근거 없는 효과 암시 없음.

ROLE

해당 PAGE_NO의 ROLE_ID·INFO_ID, 필수 모듈과 시각 증거가 실제 화면에 보임.

SOURCE LINEAGE

제품·수치·실측·실사가 승인 RAW·Proof와 일치하고, 생성 장면이나 REF_*를 원본 증거처럼 표현하지 않음.

UNIQUENESS

PRIMARY_FACT, H1 논리, 제품 컷, 모델·의상·포즈, 지배 레이아웃이 다른 장과 겹치지 않음.

INFORMATION GAIN

한 장을 가렸을 때 구매자가 잃는 독점 정보가 있음. 없다면 병합 또는 삭제.

재시도 원칙: 실패한 장만 보존 후 재생성합니다. 정적 텍스트는 image_gen 직접 생성 2회가 모두 실패한 뒤에만 번호 질문으로 승인 카피 FIXED_TEXT_LAYER 대체 여부를 확인합니다. 제품·레이아웃 오류는 후편집으로 통과시키지 않습니다.

CAPTURE_MODE

REAL_TEST는 실제 물리 결과, REAL_DEMO는 보이는 구조·사용법, AI_ILLUSTRATION은 이해용 연출입니다. 통과 후보가 없으면 NO_PROOF_HOOK으로 둡니다.

TOP1 하드 게이트

고유 소구 선명도와 반증 가능한 증명 정도인 실증 후킹력이 각각 12점 이상이어야 합니다. AI_ILLUSTRATION의 실증 후킹력은 최대 5점이며 TOP1이 될 수 없습니다.

TOP3 스토리보드

CAPTURE_MODE, 한정 주장, 시험 조건·통제, 보이는 판정, 실사 원본·편집 내역 또는 AI 표시와 타임라인·QA를 기록합니다.

고정 폰트 레이어

모션 플레이트와 카피를 분리합니다. 승인 카피는 실제 폰트 레이어로 한 번 조판해 전체 클립에 합성하고, 제품 물리 라벨은 원본 텍스처로 트래킹합니다.

실사·AI 무결성

REAL_TEST는 동일 제품·조건·판정·연속 원본을 기록합니다. REAL_TEST·REAL_DEMO 실행은 ffprobe로 RAW 클립의 decode·dimensions·duration·frames를 확인하며 도구나 값이 없으면 fail-closed합니다. ComfyUI 결과는 AI_ILLUSTRATION이며 테스트·입증 카피를 쓰지 않습니다.
