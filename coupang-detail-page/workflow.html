COUPANG DETAIL PAGE · SKILL WORKFLOW

입력부터 10장 납품까지,
한눈에 보는 실행 과정

여러 실제 상세페이지의 패턴 분석, 정보 밀도 있는 카피 매니페스트, 텍스트 포함 image_gen 생성, 장별 의미 QA, 모션·ComfyUI 핸드오프까지 이어지는 표준 흐름입니다.

10개 독립 이미지 기본 800 × 2400 한 질문씩 · 번호 응답 단계별 진행 멘트 실패한 장만 재생성
전체 흐름 온보딩 진행 멘트 리서치 카피·디자인 생성 QA 모션·ComfyUI 실행 명령

전체 흐름

각 게이트를 통과해야 다음 단계로 넘어갑니다. 기획 승인 전에는 생성하지 않고, 의미 QA가 끝나기 전에는 납품하지 않습니다.

01

한 번에 한 질문으로 입력 수집

원본, 레퍼런스, 상품 정보와 선택 사항을 번호로 확인합니다. 마지막 보기는 항상 ‘다른 답변하기’입니다.

Input gate
02

원본·레퍼런스 직접 확인

원본은 제품 정체성, 레퍼런스는 추상적 스타일 기준으로 분리합니다.

Identity gate
03

동일 제품 + 여러 상세페이지 조사

일치 근거와 출처 강도를 기록하고, 최소 3개 실제 페이지의 훅·본문·카드·캡션 패턴을 비교합니다.

Evidence gate
04

10장 카피·타이포·장면 기획

장마다 3~6개 정보 구역과 정확한 카피 매니페스트를 만들고, 같은 글자 크기 체계와 디자인 시스템을 잠급니다.

Approval gate
05

텍스트까지 image_gen에서 직접 생성

한 장씩 독립 호출하고, 제품·디자인·승인 한국어를 한 결과 안에 완성합니다.

Generation
06

TEXT · PRODUCT · LAYOUT · CLAIMS QA

직접 눈으로 검사해 실패한 장만 원인을 고쳐 다시 생성합니다.

Meaning gate
07

800 × 2400 파일 검증

비율을 안전하게 정규화하고 정확히 10장인지, 손상·중복이 없는지 검사합니다.

File gate
08

모션 기회와 ComfyUI 핸드오프

필요한 장에만 GIF·영상 아이디어를 제안하며, 실제 연결이 없으면 HANDOFF_ONLY로 표시합니다.

Optional

온보딩 원칙

한 응답, 한 질문

사용자가 이미 준 답을 다시 묻지 않습니다. 매 답변 뒤 저장한 한 항목만 확인하고 다음 질문으로 이동합니다.

선택권을 닫지 않기

추천안을 표시할 수 있지만 강요하지 않습니다. 모든 선택 질문의 마지막 번호는 언제나 다른 답변하기입니다.

작업이 보이게 만드는 진행 멘트

진행 중 업데이트 없는 상태가 60초를 넘지 않게 합니다. 각 멘트에는 완료한 것, 지금 하는 것, 다음 할 것을 담습니다.

관문별 업데이트

입력 확인 → 웹 조사 → 10장 기획 → 생성 → QA·재생성 → 정규화·납품을 지날 때마다 상태를 알립니다.

장별 생성 현황

N/10 생성 시작을 먼저 알리고, 저장·검수 후 N/10 PASS 또는 RETRY와 다음 장을 안내합니다.

실패도 즉시 공개

깨진 한글, 잘못된 제품, 작은 글자와 과장 표현을 숨기지 않고 실패 항목과 재시도 방향을 설명합니다.

짧고 검증 가능하게

막연한 “진행 중” 대신 확인한 파일 수, 조사한 페이지 수, 생성·통과한 장 수처럼 검증 가능한 상태를 말합니다.

동일·유사 제품 리서치

동일성과 출처 신뢰도를 분리하고 여러 페이지의 구조를 비교하면, ‘같아 보이는 사진’이 검증된 기능 사실로 잘못 승격되는 것을 막으면서 실제 상세페이지의 정보 밀도는 배울 수 있습니다.

판정의미사용 범위
M1 + E1/E2모델·SKU 또는 고유 라벨과 공식/신뢰 판매처가 일치명시된 사실을 카피 후보로 제안 가능
M1/M2 + E3외형과 라벨은 일치하나 마켓·개인 판매 출처외형·착용 구조 보조 확인만
M3 + E4같은 카테고리의 유사 제품문제 각도·장면·정보 구조 아이디어만
M0불일치 또는 동일 제품 확인 못함결과 없음 자체만 기록

카피·타이포·디자인을 먼저 잠그기

COPY_SYSTEM

  • 확인된 불편 또는 보이는 차이에서 후킹
  • EYEBROW·H1·BODY·CARD·CAPTION·CTA 매니페스트
  • 장마다 3~6개 정보 구역, 구역마다 한 메시지
  • 승인 카피 축약·의역·누락·추가 금지

TYPOGRAPHY_SYSTEM

  • 10장 동일한 한국어 산세리프 한 계열
  • H1 72~104px · H2 44~64px
  • BODY 32~40px · 라벨/캡션 최소 32px
  • 서체 변화·작은 각주·낮은 대비·겹침 금지

한 호출 안에서 디자인 + 카피

10장 모두 같은 시스템 블록을 반복하고, 장별 설득 목적과 장면만 바꿉니다. 이전 생성본은 다음 장의 제품 기준으로 사용하지 않습니다.

사용 사례: ads-marketing
자산: 쿠팡 모바일 상세페이지 10장 중 N장
목적과 후킹: <이 장의 역할>
입력 역할: ORIGINAL_PRODUCT / WEB_CONFIRMED_PRODUCT / STYLE_REFERENCE
제품 불변 요소: <형태·색상·부품·라벨·구성품>
공통 카피·타이포·디자인 시스템: <승인 블록>
장면·구도: 800x2400, 1:3, 3~6개 세로 정보 구역과 명확한 읽는 순서
카피 직접 조판: 승인 한국어를 완성 디자인 안에 정확히 렌더링
정확한 카피: EYEBROW / H1 / BODY / CARD·CHIP·CAPTION / CTA
타이포: H1 72~104px, H2 44~64px, BODY·라벨·캡션 최소 32px
금지: 매니페스트 밖의 글자, 누락·깨진 한글, 작은 각주, 허위 주장, 워터마크, 쿠팡 UI

이상한 결과를 잡는 4단 QA

TEXT

카피 매니페스트와 글자 단위 일치, 누락 없는 조판, 같은 서체 계열, 모든 본문·라벨·캡션 최소 32px.

PRODUCT

실루엣·색상·부품·라벨·구성품 수 일치. 임의 복제·삭제·합성 없음.

LAYOUT

후킹이 첫 시선에 보이고 3~6개 정보 구역의 순서가 명확하며 글자·제품·카드가 겹치거나 잘리지 않음.

CLAIMS

허위 기능·수치·UI·로고·페이지 번호와 근거 없는 효과 암시 없음.

재시도 원칙: 실패한 장만 보존 후 재생성합니다. 텍스트가 두 번 실패하면 임의로 후편집하지 않고 번호 선택으로 다음 경로를 확인합니다.

GIF·영상·ComfyUI

모션이 도움 되는 장면

착용 순서, 접고 펴기, 주름과 질감의 움직임, 구성품 전환처럼 정지 이미지보다 사용 이해를 높이는 장면을 추천합니다.

ComfyUI 핸드오프

시작·끝 프레임, 길이, FPS, 루프, 카메라, 입력 경로, 제품 불변 요소와 네거티브 프롬프트를 기록합니다. 연결이 없으면 HANDOFF_ONLY입니다.
