# 웹 리서치 (Phase 2)

인테이크에서 **근거가 빈 주장**이나 **최신 데이터가 필요한 지점**을 웹으로 보강한다.
리서치는 보조 단계 — 인테이크에서 충분하면 건너뛴다.

## 언제 하나 (트리거)
- 인테이크 노트의 근거가 "[리서치 필요]"로 남은 항목.
- 수치·트렌드·사실 주장인데 출처가 없을 때.
- 사용자가 "최신/요즘" 같은 시의성을 요구할 때.

## 방법
- 가벼운 확인: `WebSearch`로 후보 출처 탐색.
- 페이지 정독/스샷: `WebFetch` 또는 `browse goto <url>` → `wait --networkidle` → 필요시 `screenshot`.
- **무거운 fetch(다수 페이지) 전에는 사용자에게 "이 소스들 확인해도 될까요?" 1회 확인.**

## 소스가 많을 때 — 서브에이전트 팬아웃 (선택)
- 독립적인 소스가 3개 이상이면 Explore/general-purpose 서브에이전트에 소스별로 나눠 요약 수집.
- 각 서브에이전트는 "출처 URL + 핵심 사실 3줄"만 반환. 본문 덤프 금지.

## 출처 규칙
- 슬라이드에 들어가는 수치/주장은 **출처를 인테이크 노트에 함께 기록**(슬라이드엔 간결히, 노트엔 URL).
- 1차 자료 > 블로그. 광고/추측은 배제.
- 못 찾으면 "근거 없음"으로 표시하고 사용자에게 알린다 — 지어내지 않는다.
